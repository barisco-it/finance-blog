# Definition of Done (DoD)

A task is considered DONE only when all conditions below are satisfied.

---

## Functional Criteria

- Meets acceptance criteria defined in the issue
- No scope creep
- Works in development mode
- Works in production build

---

## Technical Criteria

- `npm run lint` passes
- `npm run build` passes
- No TypeScript errors
- No console errors in browser
- No unused imports or variables

---

## UX Criteria

- Mobile responsive
- Clean layout
- No broken links
- Consistent typography

---

## SEO Criteria (for pages)

- Unique page title
- Meta description present
- Proper heading hierarchy (H1, H2, H3)
- Clean URL slug

---

## Governance Criteria

- No unnecessary dependencies added
- No architecture drift
- Code changes are small and reviewable
- Linked to GitHub Issue

---

If any item fails, the task is NOT done.
